🛡️ Security · IDOR
Any Authenticated User Could Access Any Order

The order detail API accepted a numeric ID without checking if the requesting user actually owned the resource. Sequential enumeration exposed every transaction on the platform.

Problem
Analysis
Approach
Fix
✓ Resolved

🔍 Debugging · Data Leak
API Returning Sensitive Fields Not Requested By the Client

The user profile API was serializing entire entity objects — including email, phone, and hashed passwords — regardless of what the frontend actually requested. A silent data exposure hiding in plain sight.

Problem
Analysis
Approach
Fix
✓ Resolved
Outcome
DTO projection layer. API returns only explicitly declared fields.
⚡ Performance · N+1 Query
Product Listing Page Taking 8+ Seconds to Load

The catalogue page was fast in development (30 products) and catastrophically slow in staging (2,000 products). An N+1 query pattern hiding behind lazy loading was executing a separate DB call per product.

Problem
Analysis
Approach
Fix
✓ Resolved
Outcome
JOIN query + pagination. 8.2s → 340ms. 96% faster.
